# Reference-Implementation Roadmap

The canonical documents are complete enough for a pilot. The next work should test the method rather than expand it.

1. Choose the public project name and verify namespace and trademark availability.
2. Choose the license; Apache-2.0 for the whole repository is the simplest strong default, while separate documentation and code licenses are possible.
3. Publish a private security-reporting channel and maintainer model.
4. Extend the neutral Rust CLI from manifest/reference validation, inspection, context packets, and conformance reports into composition and compatibility checks.
5. Deepen the Rust binding from Cargo/package checks into source-level dependency-boundary enforcement, then implement the Swift binding.
6. Publish one realistic reference application, not only toy examples.
7. Replace one implementation with another and document contract, state, and operational compatibility.
8. Maintain part of the reference application primarily through agents using the canonical skills.
9. Measure context size, invalid-change rejection, compatibility failures, recovery behavior, and agent success.
10. Revise from evidence, then publish RMS 1.0.
